🐙 Система контроля версий Git и GitHub

Управление версиями кода, совместная разработка, публикация сайтов в интернете. Лектор: Хаятова Лилия Рэхтямовна

📌 1. Что такое Git и GitHub?

📦 Git

Git — это распределённая система контроля версий. Она позволяет отслеживать изменения в коде, возвращаться к предыдущим версиям, работать над проектом в команде.

git --version # проверка установки
🌐 GitHub

GitHub — это веб-платформа для хостинга Git-репозиториев. Позволяет хранить код в облаке, делиться проектами, публиковать сайты через GitHub Pages.

github.com

⌨️ 2. Основные команды Git

🚀 Инициализация и настройка
git init # создать репозиторий
git config --global user.name "Ваше Имя"
git config --global user.email "email@example.com"
📝 Сохранение изменений
git add . # добавить все файлы
git add index.html # добавить конкретный файл
git commit -m "описание изменений"
git status # проверить состояние
git log # история коммитов
🔗 Работа с удалённым репозиторием
git remote add origin https://github.com/пользователь/репозиторий.git
git push -u origin main # отправить изменения
git pull # получить изменения
git clone https://... # склонировать репозиторий
🌿 Ветвление (branch)
git branch # список веток
git branch new-branch # создать ветку
git checkout new-branch # переключиться на ветку
git merge branch-name # слияние веток

🌍 3. GitHub Pages — публикация сайта в интернете

GitHub Pages — бесплатный сервис для хостинга статических сайтов прямо из репозитория GitHub. Идеально подходит для портфолио, лендингов, учебных проектов.

Шаг 1: Создайте репозиторий на GitHub
Шаг 2: Загрузите файлы сайта (index.html, css, img)
Шаг 3: Перейдите в Settings → Pages
Шаг 4: В разделе "Branch" выберите main (или master) и папку /root
Шаг 5: Сохраните — сайт станет доступен по адресу:
https://ваш_username.github.io/название_репозитория/
💡 Важно

Главный файл сайта должен называться index.html и находиться в корне репозитория (или в папке, выбранной в настройках). После публикации изменения появляются через 1-2 минуты.

📋 Задание к публикации

  1. Мой первый сайт — сайт-портфолио верстальщика, созданный в практической работе по теме №13.
  2. Сайт-визитка — краткая информация о себе, навыках, контактах (можно создать отдельную страницу или использовать уже готовый проект).

📝 Пошаговая инструкция публикации

  • Зарегистрируйтесь на GitHub.com (если ещё нет аккаунта).
  • Создайте новый репозиторий (кнопка «New»), назовите его, например, my-first-site.
  • Настройте Git на компьютере (установите с git-scm.com, если ещё не установлен).
  • Откройте терминал (командную строку) в папке с вашим проектом «Мой первый сайт».
  • Выполните команды:
    git init
    git add .
    git commit -m "Первый коммит: мой первый сайт"
  • Свяжите локальный репозиторий с удалённым:
    git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
  • Отправьте код на GitHub:
    git push -u origin main
    (если основная ветка называется master, используйте git push -u origin master)
  • В репозитории на GitHub перейдите в Settings → Pages.
  • В разделе "Branch" выберите main (или master), оставьте папку /(root), нажмите Save.
  • Через минуту сайт будет доступен по адресу: https://ВАШ_ЛОГИН.github.io/НАЗВАНИЕ_РЕПОЗИТОРИЯ/
  • Аналогично опубликуйте второй проект — «Сайт-визитку» (в отдельном репозитории).

✅ Результат работы — две активные ссылки на опубликованные сайты, которые можно отправить преподавателю или добавить в портфолио.

💡 6. Советы по работе с Git и GitHub

📝 Правила хороших коммитов
  • Пишите осмысленные сообщения коммитов на русском или английском.
  • Коммитьте логические завершённые изменения.
  • Не коммитьте большие бинарные файлы (картинки лучше оптимизировать).
  • Используйте .gitignore, чтобы исключить ненужные файлы (node_modules, .DS_Store).
🤝 Совместная работа
  • Pull Request — предложение изменений в чужой репозиторий.
  • Fork — копия репозитория для самостоятельной работы.
  • Clone — скачивание репозитория на компьютер.
