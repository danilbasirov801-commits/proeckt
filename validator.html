✅ Валидность кода HTML

Проверка на соответствие стандартам W3C. Лектор: Хаятова Лилия Рэхтямовна

📌 1. Зачем нужна валидация кода?

Валидный HTML-код — это код, написанный по спецификации W3C (World Wide Web Consortium), в которой собраны стандарты и рекомендации по удобству и универсальности Всемирной сети.

📖 Аналогия

Правила валидного HTML похожи на правила русского языка. Если вы не закроете тег (как не закончите абзац текста) — нарушится структура и смысловая составляющая.

🎯 Почему это важно?
  • ✅ Браузеры лучше обрабатывают код
  • ✅ Страницы быстрее загружаются
  • ✅ Поисковые системы верно понимают смысл тегов
  • ✅ Другим разработчикам легче читать код
<!-- НЕВАЛИДНЫЙ код (закрывающий тег потерян) -->
<p>Текст без закрывающего тега

<!-- ВАЛИДНЫЙ код -->
<p>Правильный абзац с закрытием</p>

🔧 2. Валидатор W3C — главный помощник

Валидатор W3C — это онлайн-инструмент, который автоматически проверяет HTML-код на соответствие стандартам. Используется повсеместно, так как создан теми, кто разрабатывает спецификации языка.

📥 Три способа проверки

🌐 По URL

Введите адрес сайта, который уже опубликован в интернете.

📁 Загрузить файл

Отправьте HTML-файл с компьютера.

📝 Вставить код

Скопируйте и вставьте код в специальное поле.

🔍

The W3C Markup Validation Service

https://validator.w3.org/

🚀 Открыть валидатор

📊 Что проверяет валидатор?

Валидатор оценивает синтаксическую составляющую кода: проверяет пропущенные или ошибочные теги, верное закрытие блоков, вложенность.

⚠️ Предупреждения

Незначительные неточности в коде, которые не сломают сайт, но не соответствуют стандартам. Например, рекомендация добавить атрибут type для стилей.

❌ Ошибки

Серьёзные проблемы, которые могут повлиять на работу кода. Например, неверная вложенность тегов, незакрытые элементы, отсутствие обязательных атрибутов (alt, title).

⚠️ 3. Типичные ошибки в HTML

🔴 Незакрытые теги
<p>Текст без закрытия
🔴 Отсутствие alt у <img>
<img src="photo.jpg">
🔴 Неверная вложенность
<strong><em>Текст</strong></em>
🔴 Пустой <title>
<title></title>
<!-- Пример фрагмента с ошибками, который проверим на валидаторе -->
<!DOCTYPE html>
<html>
<head>
  <title></title> <!-- Ошибка: пустой title -->
</head>
<body>
  <h1>Заголовок
<!-- Ошибка: незакрытый h1 -->
  <img src="logo.png"> <!-- Ошибка: нет alt -->
</body>

💡 4. Плагины-хинтеры для редакторов

Помимо классического валидатора, существуют хинтеры — плагины для редакторов кода, которые подчёркивают ошибки прямо при написании кода.

🔌 HTMLHint для VS Code

Плагин, который подсвечивает ошибки жёлтой волнистой линией прямо в редакторе. Позволяет видеть и исправлять проблемы мгновенно.

<p>Текст с незакрытым тегом
<img src="photo.jpg" alt="">

🟡 Жёлтая волнистая линия — сигнал о проблеме

📦 Другие полезные инструменты
  • ESLint — для JavaScript
  • Stylelint — для CSS
  • Prettier — автоформатирование
# Установка HTMLHint в VS Code:
Ctrl+Shift+X → поиск "HTMLHint" → установить

📌 Практическое задание

Проверьте статью, которую вы выполняли на одном из прошлых занятий, на валидаторе W3C.

📄 Задание 1

Возьмите любую ранее созданную HTML-страницу (например, статью или страницу со ссылками).

🔍 Задание 2

Откройте validator.w3.org и проверьте код (через загрузку файла или прямой ввод).

✏️ Задание 3

Исправьте все ошибки и предупреждения, которые найдёт валидатор. Повторите проверку до чистого результата.

Дополнительное задание: Установите плагин HTMLHint в VS Code и убедитесь, что ошибки подсвечиваются прямо при наборе кода.

🌐 Открыть валидатор W3C 🔌 Установить HTMLHint

✅ Результат: после исправления ошибок сохраните страницу и убедитесь, что она проходит валидацию. Чистый валидный код — признак профессионализма!

📚 5. Рекомендации по поддержанию чистоты кода

  • Всегда закрывайте теги, даже одиночные (<img />, <br /> — в XHTML, в HTML5 необязательно, но для единообразия полезно).
  • Добавляйте атрибут alt для всех изображений — это важно для доступности и SEO.
  • Следите за правильной вложенностью: вложенные теги должны закрываться в обратном порядке.
  • Используйте семантические теги (<header>, <main>, <article>) вместо универсальных <div> где это возможно.
  • Перед публикацией сайта всегда прогоняйте его через валидатор.
  • Настройте автоформатирование (Prettier) и хинтер (HTMLHint) в редакторе кода.
